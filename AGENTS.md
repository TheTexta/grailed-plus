## UI Styling Policy

- For any new UI component, always implement styling in `src/ui.css`.
- Reuse existing classes, variables, and patterns in `src/ui.css` before introducing anything new.
- Do not add custom CSS in other files (including inline styles, style tags, or new stylesheet files) without explicit user permission first.
- If custom CSS appears necessary, stop and ask for permission before making those changes.


## Changes that affect functionality
- always ensure docs are updated after functionality is updated
