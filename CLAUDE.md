## Design Context

### Users
- Primary users: buyers and sellers on Grailed.
- Context: in-flow usage while browsing listing pages and evaluating item value quickly.
- Core job-to-be-done: find better deals for marketplace items with fast, decision-ready signals (pricing trends, comparisons, seller context).

### Brand Personality
- 3-word personality: clean, minimalist, black/white.
- Emotional goal: confidence first.
- Voice/tone implication: factual, concise, utility-first; avoid decorative excess.

### Aesthetic Direction
- Visual direction: technical and data-dense, while preserving minimalist structure.
- Product reference: grailed.com (fit in naturally with the host environment).
- Theme support: both light and dark mode are required from existing product behavior.
- Palette intent: black/white dominant with restrained accent usage only for state clarity.

### Design Principles
1. Confidence Through Clarity
- Prioritize legible data hierarchy and unambiguous state messaging over decorative styling.

2. Native-to-Grailed Integration
- UI should feel like a natural extension of grailed.com; avoid visual motifs that look third-party.

3. Minimal Surface, Maximum Signal
- Keep layouts compact and clean, but preserve high information density where it improves decisions.

4. Deterministic Theming
- Light/dark behavior must be predictable and token-driven, with one clear source of truth for component states.

5. Utility-First Interactions
- Every interaction should reduce evaluation time (fewer clicks, immediate feedback, practical metadata/comparison access).
